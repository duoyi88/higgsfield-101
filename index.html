01

What is Higgsfield? (dumb version)

One workspace. Many AI video models. Extra director tools on top.

Explain like I am five

Higgsfield is not one magic model. It is a video mall: Kling, Seedance, Veo, Wan, MiniMax, and more live under one login. You pick the tool for the shot. Then you stack Higgsfield’s own director toys: camera control, character lock, lip-sync, motion copy.

→
5–15s clip
→
Edit / stitch

Long videos = many short clips + a real editor. Not one giant generate button.

Deep dive: platform map

What you get in one place

  • Text-to-video and image-to-video
  • Video restyle / reshoot from a reference clip
  • Cinema Studio (virtual cameras & lenses)
  • Soul ID (same face across shots)
  • LipSync, motion control, first/last frame lock
  • MCP / CLI / Skills so agents can generate for you

What it is not

  • Not a full NLE like Premiere
  • Not a free unlimited render farm
  • Not perfect long-form identity lock without prep
  • Not “type one sentence, get a 5-minute film”

Deep dive: step-by-step image-to-video
  1. Write one shot only

    Subject + action + camera + light. Example: “Medium close-up, young Taiwanese woman waves at camera, handheld iPhone feel, soft daylight.”

  2. Generate or upload a key still

    Face sharp, outfit fixed, background readable. Reject plastic skin and weird hands before you animate.

  3. Animate with a short duration

    Start at 5–8 seconds. One gesture. No multitasking.

  4. Pick the model for the job

    People talking / walking → Kling. Wide outdoor mood → Veo. Multi-shot + audio → Seedance. Drafts → MiniMax.

  5. Export and stitch outside

    CapCut, Premiere, DaVinci. Add captions, BGM, logo sting.

04

Strengths & honest limits

Use the strengths. Design around the limits. No fantasy.

Strengths

  • Many top models under one plan
  • Real camera / lens control (Cinema Studio)
  • Character consistency via Soul ID
  • Mature image-to-video workflow
  • Agent-ready MCP / Skills / CLI
  • Social-first formats (9:16, short ads, UGC)

Limits

  • Clips are short (often 5–15s)
  • Long motion drifts: faces, hands, continuity
  • Human close-ups need more control than landscapes
  • Credits vary wildly by model
  • Strict moderation on some models
  • Option overload burns money if you have no plan
05

MCP & Skills: generate from your agent

Stay in chat. Let the agent pick models, poll jobs, and return URLs.

Dumb version

MCP = plug Higgsfield into Claude / OpenCode / Cursor. Skills = reusable recipes (generate, soul character, product shoot). You talk. The agent submits jobs and waits.

A. MCP (hosted)

https://mcp.higgsfield.ai/mcp

  • OAuth with your Higgsfield account
  • No API key copy-pasta
  • Common tools: generate image/video, create character, poll status
# Claude Code example
claude mcp add --transport http --scope user \
  higgsfield https://mcp.higgsfield.ai/mcp

B. Skills (workflow packs)

npx skills add higgsfield-ai/skills
higgsfield auth login
  • generate — general image/video
  • soul — train / reuse a character
  • product-photoshoot — product/marketing packs

Also useful: video → prompt reverse, product URL → launch clip, long video → vertical shorts.

Deep dive: OpenCode config gotcha

OpenCode uses servers, not mcpServers. Put this in ~/.config/opencode/opencode.json or project MCP config:

{
  "mcp": {
    "higgsfield": {
      "type": "remote",
      "url": "https://mcp.higgsfield.ai/mcp"
    }
  }
}

Exact key names vary by client. After adding, fully restart the app and complete OAuth on first tool call.

Prompt recipe that works in chat (MCSLA)

  1. Model — Kling / Seedance / Veo…
  2. Composition — frame, angle, subject placement
  3. Subject — concrete wardrobe, age vibe, props
  4. Lighting — golden hour, soft window, neon night
  5. Aesthetic — handheld vlog, 35mm, light grain

Recipe for good assets

Still first. Character second. One action per clip. Cheap drafts, expensive finals.

1Lock characterSoul ID / refs
2Make key stillshero / walk / talk
3Animate short5–8s I2V
4Pick winnerskill AI-looking takes
5Stitch + captioneditor + BGM

Do

  • One action per clip
  • Reuse the same still family
  • Write captions before lip-sync
  • Keep wardrobe and hair fixed per episode
  • Iterate drafts on cheap models

Don’t

  • “Walk, talk, wave, turn, change outfit” in one prompt
  • Animate a bad still and hope video fixes it
  • Switch face style mid-series without Soul ID
  • Spend Seedance credits on throwaway tests
  • Skip post: raw AI clips look unfinished
Deep dive: agent chat script for batch assets
  1. Train Soul character “Maya” from 12–20 clean reference photos.
  2. Generate 6 key stills in Taipei street / café / night market.
  3. Animate each still to 6s with Kling, handheld, one action.
  4. Return only the top 4 clips + stills as a package.
  5. Human picks, then editor stitches to 30s with captions.

80% prep. 20% generate. This is the checklist that stops face drift and credit waste.

Consent first. If the face is a real person, get written permission for commercial use. Fully fictional characters are simpler legally.

  • Ratio 9:16
  • Persona age vibe, energy, wardrobe
  • Scenes 2–3 max (street / market / café)

Audio plan

Caption ZH idea:「嗨，我是 Maya，每週帶你走台北小吃。」

VO / line 2: “Tonight: night market classics in under two minutes.”

End CTA: “Follow for the next stop.”

Keep spoken lines short. AI mouths fail on monologues.

08

MVP: ship a 20–30s intro today

Minimum path. No perfectionism. Re-render only the failed shot.

  1. 1

    Write 3-line persona + 2 scenes

    Who she is, what she wears, where we are. Done in five minutes.

  2. 2

    Generate 8–12 stills, pick 1 master face

    Optional: train Soul ID if this becomes a series.

  3. 3

    Build a 5-row shot list

    One action each. Times add to ~30s.

  4. 4

    I2V every row at ~6 seconds

    Kling for people. Cheap model for empty plates.

  5. 5

    Stitch in CapCut

    Captions, light grade, BGM, end card. Publish.

Remember

  • Higgsfield = multi-model studio, not one model
  • Image → video is the people path
  • Soul ID saves series consistency
  • MCP/Skills multiply output once setup is done
